# Maintainers

## Roles and Responsibilities

| Role | Responsibility |
|---|---|
| **Foundation Director** | Strategic vision, external representation, final call in a genuine deadlock |
| **Documentation Maintainer** | Reviews and approves changes to `docs/` |
| **Security Maintainer** | Owns `SECURITY.md`, security workflows, incident response |
| **Brand Maintainer** | Approves changes to `assets/branding/` and brand documents |
| **Infrastructure Maintainer** | Manages GitHub Actions workflows and repo configuration |

## Becoming a Maintainer

1. Make several high-quality contributions over a period of time
2. Show a solid understanding of CeloHT's vision and values
3. Get nominated by a current maintainer
4. Get approval from the Maintainer Council

See [`CODEOWNERS`](../CODEOWNERS) for specific responsibility by file path.
